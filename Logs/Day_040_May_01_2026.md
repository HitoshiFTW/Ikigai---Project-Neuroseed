## 1. Objective

Day 39 completed the cognitive scaffold: state abstraction, two-scale prediction, cross-level conflict detection, and a six-layer escalation stack converting conflict into graded behavioral response. The Day 39 verdict identified the next missing capability as directed investigation -- choosing what to explore rather than how much. Pursuing that capability immediately, however, would put behavioral mechanism on top of an unproven representational substrate. The system's internal state remains a flat dictionary of cluster IDs, transition counts, and scalar signals; it has no compositional representation of "this state plus this action" as a single object that can be stored, compared, transformed, or decomposed back into parts. Without such a substrate, any future planning, hypothesis generation, or sequence reasoning would be expressed as ad-hoc dictionary manipulations that do not generalize across contexts.

Day 40 establishes the internal representation layer of the system. The chosen substrate is Vector Symbolic Architecture (VSA): high-dimensional binary hypervectors operated on by exclusive-or binding, majority-vote bundling, and hamming/cosine similarity. The strategic plan from the May 1 research direction document selects VSA as the prerequisite for both the Options Framework (Day 41-44) and Active Inference (Day 45-48) phases, on the grounds that VSA reuses existing Ikigai population sizes (≈400 neurons → 400-dim hypervectors) and adds zero incremental memory overhead. Day 40 is split into four packs that build the representation stack from primitives to manipulation: Pack 1 introduces the operators and item memory; Pack 2 wires per-tick (cluster, action) encoding into a fixed circular buffer with multi-scale similarity tracking; Pack 3 validates that the resulting signals encode real behavioral structure rather than statistical artifacts; Pack 4 demonstrates that the system can decompose what it builds and compose short sequences. All four packs preserve strict zero behavioral drift: every tick's action, routing, and top_confidence under flag = ON is bit-identical to flag = OFF when run with the same seed. This is enforced by structural separation: the VSA hook reads from but never writes to scoring, routing, or any K-modulated variable.

## 2. Implemented Changes

**Pack 1 -- VSA Foundation (default OFF, storage-only)**

The minimum substrate for Vector Symbolic computation. Five new IkigaiContext fields and four methods are added to `ikigai/core/state.py`: `enable_vsa` (bool, default False), `vsa_dim` (int, default 400, matching cortical population scale), `vsa_items` (dict, name → bytearray of 0/1 of length vsa_dim), `vsa_current` (bytearray or None, lPFC working register). Methods: `vsa_init_items(names, seed=42)` lazily populates `vsa_items` with deterministically-generated near-orthogonal binary hypervectors via a seeded `random.Random(seed).getrandbits(1)` call per bit (idempotent: returns immediately if `vsa_items` is non-empty); the default name set is the five base concepts (A, B, STATE, ACTION, CONTEXT) plus the Pack 2/3/4 vocabulary discussed below. `vsa_bind(a, b)` returns the element-wise XOR of two bytearrays as a new bytearray (O(N), self-inverse: bind(bind(A, B), A) == B). `vsa_bundle(vectors)` returns the per-position majority vote thresholded at strictly greater than half the input count (O(N×K) for K input vectors). `vsa_similarity(a, b)` returns the hamming match ratio (identical = 1.0, random ≈ 0.5). The Pack 1 implementation is purely structural: no per-tick activity, no scoring path, no behavioral coupling. The Pack 1 validation harness initializes items, builds C = bind(A, B), recovers B = bind(C, A), and verifies that recovered B is bit-exact equal to original B (similarity = 1.0000 across 400 dimensions). Noise robustness is verified by flipping 5% then 10% of bits in C and confirming recovery similarity remains above 0.9 in both cases.

**Pack 2 -- Observational VSA Encoding (default OFF, fixed buffer)**

Per-tick translation of system state into the VSA substrate, with multi-scale stability tracking. Twelve new IkigaiContext fields: `vsa_event` (current event vector, default None), `vsa_prev_event` (previous event vector, default None), `vsa_buffer_size` (int, default 128), `vsa_buffer` (list of 128 None slots, circular), `vsa_index` (int, default 0, write head), `vsa_count` (int, default 0, total events recorded), and three running aggregate triples for sim_1, sim_5, sim_20 means (`vsa_sim1_sum`, `vsa_sim1_count` and parallels). Three new methods: `vsa_cosine(a, b)` returns the centered cosine similarity for ±1-mapped binary vectors (= 2 × hamming_match - 1; identical = +1, random ≈ 0, anti-correlated = -1); `vsa_encode_event(cid, action_type)` builds E = bind(CID_<cid>, ACT_<action_type>) using items from `vsa_items`, returning None if either component is missing; `vsa_record_event(event, ...)` is the per-tick entry point that updates aggregates and appends to the buffer. The buffer is a fixed-size circular: `vsa_buffer[vsa_index] = event; vsa_index = (vsa_index + 1) % 128; vsa_count += 1`. Multi-scale similarity is computed before the append: sim_1 = cosine(event, vsa_event); sim_5 = mean cosine of event against the last 5 events in the buffer; sim_20 = mean over the last 20 events. The aggregates accumulate sums and counts; mean accessors are exposed as `@property` decorators. The item vocabulary is expanded by Pack 2 from the five Pack 1 base concepts to twenty-seven items: 5 base + CID_0 through CID_15 (sixteen cluster identity vectors) + ACT_approach, ACT_explore, ACT_wait, ACT_idle_recover, ACT_edit_code, ACT_run_experiment (six action vectors covering both raw BG action types and the Day 36 exec_diversity-mapped variants). The 27 items × 400 bytes = 10.8 KB is the fixed item memory budget; the 128-slot buffer holds at most 128 × 400 = 51.2 KB of event vectors. Total Pack 2 storage is bounded above by ≈62 KB regardless of run length.

The ikigai.py modification is a single conditional injection at the post-action region (line 34655 in the current source), inside the existing Day 39 Pack 3 cluster-transition block that already has `_ct_curr_cid` (current cluster ID) in scope. The injection sequence: snapshot `_vsa_prev_cid = _IKG._cluster_prev_id` BEFORE the assignment to current cid (necessary for Pack 3 classification, see below); execute the existing `_IKG._cluster_prev_id = _ct_curr_cid` assignment; then if `_IKG.enable_vsa`, lazily call `vsa_init_items()` if items are empty, encode the current event via `vsa_encode_event(_ct_curr_cid, last_action_type)`, and record it via `vsa_record_event` with classification arguments. The hook is gated by the enable flag at every step; when the flag is False, the hook body is skipped entirely. The pack does not modify BG scoring, routing, action selection, top_confidence, or any K-modulated variable. The buffer length is fixed at construction; no growing structures.

**Pack 3 -- Semantic Validation (default OFF, signal only)**

Verifies that VSA spikes and similarities encode real behavioral structure rather than vector-space artifacts. Twelve new IkigaiContext fields: `vsa_spike_threshold` (float, raised from Pack 2's 0.3 to 0.5 for semantic coherence), `vsa_spike_count` (canonical Pack 3 spike counter, replacing Pack 2's `vsa_sim_spikes` which is retained as a back-compatibility alias); spike classification counters `vsa_spike_cluster_change`, `vsa_spike_action_change`, `vsa_spike_both_change`; same/different cluster sim sums `vsa_same_cluster_sim_sum`/`count`, `vsa_diff_cluster_sim_sum`/`count`; pattern reuse `vsa_pattern_match_count`, `vsa_pattern_total`, `vsa_pattern_threshold` (default 0.8); and `vsa_prev_action` for action-change detection. The `vsa_record_event` method signature is extended to optionally accept (curr_cid, prev_cid, curr_action, prev_action) keyword arguments. When provided, the method classifies each spike: cluster_changed = (prev_cid != curr_cid), action_changed = (prev_action != curr_action), and increments the corresponding counters; same/diff cluster similarity sums accumulate sim_1 separately for within-cluster vs across-cluster transitions; pattern reuse scans the buffer counting entries with similarity above 0.8 to the current event, accumulating `vsa_pattern_match_count` and `vsa_pattern_total`. The classification and pattern-reuse computation is signal-only: no path from these counters to scoring or routing.

The ikigai.py hook is updated to pass the four classification arguments to `vsa_record_event`. The critical structural change is the snapshot ordering: `_vsa_prev_cid = _IKG._cluster_prev_id` must execute BEFORE the line `_IKG._cluster_prev_id = _ct_curr_cid` overwrites the field; otherwise Pack 3 would see prev_cid == curr_cid for every tick and miscount cluster changes as zero. The Pack 2 hook is therefore reordered: snapshot first, then perform the existing cid-prev assignment, then encode and record. This reordering is correct under all enable flags: the snapshot has no side effects when `enable_vsa = False`.

**Pack 4 -- Representation Manipulation (default OFF, signal only)**

Demonstrates that the system can decompose bound events back into components and compose short sequences. Six new IkigaiContext fields: `vsa_recover_act_sim` (cumulative similarity of recovered action vector to original), `vsa_recover_cid_sim` (cumulative similarity of recovered cluster vector), `vsa_recover_count` (sample count); `vsa_chain_event` (last composed chain vector), `vsa_chain_sim` (cumulative chain coherence sum), `vsa_chain_count` (sample count). Two new methods: `vsa_test_recovery(event, cid_vec, act_vec)` performs both decompositions -- recover_act = bind(event, cid_vec); recover_cid = bind(event, act_vec) -- and accumulates cosine similarities of each recovered component against the original. Because XOR is exact and self-inverse, when event = bind(cid_vec, act_vec) was constructed cleanly, recover_act = act_vec exactly and recover_cid = cid_vec exactly; similarity is mathematically 1.0 in every clean invocation. `vsa_compose_chain(e_curr, e_prev)` builds chain = bundle([e_curr, e_prev]) using the existing 2-vector bundle (which under the strict-majority threshold > N/2 reduces to bitwise AND for N=2), stores the result in `vsa_chain_event`, and accumulates the average of cosine(chain, e_curr) and cosine(chain, e_prev) into `vsa_chain_sim`. Three `@property` accessors are added: `vsa_recover_act_mean`, `vsa_recover_cid_mean`, `vsa_chain_mean`.

The ikigai.py hook is extended to capture `_vsa_prev_for_chain = _IKG.vsa_event` BEFORE `vsa_record_event` mutates the field (analogous to the Pack 3 prev_cid snapshot), then call `vsa_test_recovery(event, cid_vec, act_vec)` with the corresponding items pulled from `vsa_items` by name, and `vsa_compose_chain(event, prev_event)` using the snapshotted previous event. Both Pack 4 methods are no-ops when their inputs are None (first tick when no previous event exists; missing items in vocabulary). Storage growth is zero: chain_event overwrites a single bytearray slot; recovery and chain sums are scalar accumulators.

## 3. System Behavior (Post-Change)

With all Day 33 through Day 40 layers active under seed = 42 over 1500 waking ticks, the system operates a five-stage pipeline per waking tick: (1) Day 38 Pack 1-2 computes state-level transition error and prediction confidence on coarse state keys; (2) Day 39 Pack 3-4 computes cluster-level error and confidence on cluster IDs; (3) Day 39 Pack 5 computes cross-level conflict and signed conflict; (4) Day 39 Packs 6-11 fire conditionally during attention windows; (5) Day 40 Pack 2-4 encodes the (cid, action) event into a 400-dim hypervector, records it in the circular buffer, classifies any sim_1 < 0.5 spike by what changed, and tests bind/bundle reversibility.

The Pack 2 multi-scale similarity hierarchy is empirically present: sim_1 mean = 0.8586 (most consecutive ticks have nearly-identical (cid, action) → identical events), sim_5 mean = 0.8408, sim_20 mean = 0.7962. Variance ordering is strictly preserved: var(sim_1) = 0.1120 > var(sim_5) = 0.0598 > var(sim_20) = 0.0394 over a 1500-tick run with sim_1 sampled per tick (1259 samples after the warmup tick filtering). The variance ratio of sim_20 to sim_1 is 0.352 -- the 20-tick rolling window smooths fluctuations to roughly one-third the per-tick variance, confirming the multi-scale structure operates as designed.

The Pack 3 signal classification produces unambiguous results: in 1500 ticks, 185 sim_1 spikes (events where similarity dropped below 0.5) were detected, of which 100% had at least one underlying change (cluster change, action change, or both). The structural property is enforced by XOR: if (cid, action) is identical to the previous tick, the bound event is bit-identical and sim_1 = 1.0 (no spike possible). Of the 185 spikes, 56 (30.3%) coincided with a cluster change, 185 (100%) coincided with an action change, and 56 (30.3%) coincided with both. Action-change is therefore the dominant proximate cause of spikes, with cluster-change as a frequent co-occurrence; pure cluster-change spikes (action stayed but cluster shifted) are essentially absent in this regime, consistent with the Pack 39 finding that cluster transitions usually accompany action transitions in the steady state.

The Pack 3 cluster separation analysis produces a +0.948 separation gap: same-cluster transitions (1203 samples, prev_cid == curr_cid) have mean sim_1 = 0.9008; different-cluster transitions (56 samples) have mean sim_1 = -0.0471. This is the strongest single semantic property of the VSA layer: distinct cluster identities map to near-orthogonal hypervectors (mean sim ≈ 0 for different-cluster pairs is the signature of orthogonality in the centered cosine metric), while shared cluster identities map to near-identical hypervectors. The separation is not engineered: it follows from the random-orthogonal initialization of CID_i vectors, the XOR composition with action vectors, and the structural fact that same-cluster events binding identical cluster vectors with identical action vectors produce identical bound events.

Pattern reuse is high: 97095 buffer matches over 1259 events (mean 77.1 matches per event, threshold sim > 0.8) -- on a 128-slot buffer, this means roughly 60% of buffered events are above-threshold matches for the current event. The Pack 4 decomposition is exact: recover_act and recover_cid both have similarity = 1.0000 to their original components across all 1260 samples, with zero variance and zero degradation across the run. Chain coherence (the mean of cosine(chain, e_t) and cosine(chain, e_t-1)) is 0.9293, well above the 0.6 specification threshold; this value matches the prediction from the spike rate (≈86% stable ticks where chain = identity → cosine = 1.0; ≈14% transition ticks where 2-vector AND-bundle of distinct vectors gives cosine ≈ 0.5; weighted mean ≈ 0.93).

Behavior under flag = ON is bit-identical to flag = OFF at the same seed. Action distribution: edit_code = 42.09%, idle_recover = 41.95%, run_experiment = 15.96%; routing fallback_bg = 97.93%; top_confidence mean = 0.5047. These values match Day 39's seeded baseline exactly. Every one of the 1447 waking ticks produced identical (atype, routing) and bit-identical (delta < 1e-9) top_confidence between ON and OFF runs.

## 5. Key Findings

- **XOR binding is exact in practice and remains exact at scale.** The VSA literature treats XOR self-inverse (bind(bind(A, B), A) == B) as a mathematical guarantee, but mathematical guarantees do not survive contact with real software unless every step preserves them. Pack 4's 1260 in-loop recovery tests verify that the binding operator, the per-tick item lookup, and the subsequent recovery via second binding all preserve bit-exact identity. Recovery similarity is 1.0000 with zero variance across the entire 1500-tick run, including ticks where the underlying (cid, action) is changing rapidly and where the system is in active conflict-attention windows. This is the load-bearing structural property of the VSA layer: any future computation that constructs a bound representation can be confident the components remain extractable. Without this property, planning, hypothesis representation, and sequence reasoning all become unreliable -- a forward model that cannot decompose its own predictions cannot reason about the constituents of those predictions.

- **Cluster geometry transfers into hypervector space without engineering.** The +0.948 separation gap between same-cluster and different-cluster transition similarities is the single most striking result of Day 40. Same-cluster transitions (events where prev_cid == curr_cid) average sim_1 = +0.9008 -- the bound events are nearly identical because both cluster identity vectors and (often) action vectors are unchanged. Different-cluster transitions average sim_1 = -0.0471 -- statistically indistinguishable from zero, the signature of orthogonality in the centered cosine metric. The orthogonality is not a designed property; it is a consequence of randomly-initialized CID_i vectors having ≈50% bit overlap with each other (orthogonal in expectation) combined with XOR composition that preserves orthogonality of the prefix when the suffix changes. The system's clustering geometry, originally encoded in scalar cluster IDs that have no spatial relationship to each other, is preserved when the scalar IDs are mapped into 400-dimensional binary hypervectors. This means any future operation that compares events by cosine similarity is effectively comparing them by cluster membership: events from the same cluster are similar, events from different clusters are not. This is precisely the property that makes VSA suitable as a substrate for content-addressable memory and for cosine-based "cleanup" decoding in decision pipelines.

- **Multi-scale similarity preserves time-scale structure.** The variance ordering var(sim_1) > var(sim_5) > var(sim_20) is not a calibration target but a structural property of rolling-average smoothing applied to a process with finite autocorrelation. The empirical ratios are var(sim_5)/var(sim_1) = 0.534 and var(sim_20)/var(sim_1) = 0.352. For an idealized process with autocorrelation length τ, the variance of the k-step rolling mean would be reduced by a factor approximately k/τ for k >> τ, or approximately 1 + k/(2τ) in the intermediate regime. The observed reductions (0.534 at k=5, 0.352 at k=20) are consistent with an autocorrelation length of roughly 4-5 ticks, matching the typical inter-spike interval observed during conflict-attention windows. This means the multi-scale measurements correctly distinguish transient (single-tick) variation from sustained regime differences (multi-tick): a sim_1 spike at a given tick is a discrete event signal; a sustained drop in sim_5 is a regime-change signal; a sustained drop in sim_20 is a baseline-shift signal. The three measurements together form a temporal hierarchy of signal types, each with its own time scale and use case.

- **2-vector bundling under strict-majority threshold reduces to bitwise AND.** The vsa_bundle implementation thresholds the per-position sum at strictly greater than half the input count. For the chain composition case (N = 2), this means a bit is set in the output only when both inputs have the bit set -- the bitwise AND. This is a specific and not-fully-symmetric choice: it biases the chain toward the intersection of the two events rather than their union or a balanced average. The empirical consequence is that during stable regimes (where consecutive events are identical), chain = E_t = E_{t-1}, sim(chain, E_t) = sim(chain, E_{t-1}) = 1.0; during transitions (where the two events are distinct), chain has roughly 25% bit density (from 50% × 50% AND), and sim(chain, E_t) ≈ sim(chain, E_{t-1}) ≈ 0.5 in centered cosine. The mean chain coherence across the run, 0.9293, is the weighted average of these two regimes: 86% stable × 1.0 + 14% transition × 0.5 ≈ 0.93. This matches observations exactly. For Day 41+ work involving longer chains (3-5 events bundled), the strict-majority threshold becomes meaningfully different from AND -- a 3-vector bundle requires 2-of-3 majority, which produces balanced sim ≈ 0.5 to each component. Day 40's 2-vector chain is an intentionally narrow first test of composition; longer chains will be addressed when planning and option signatures arrive.

- **VSA item budget is tiny relative to total system memory.** The full Day 40 VSA layer consumes ≈62 KB at peak: 27 items × 400 bytes (10.8 KB) plus 128 buffer slots × 400 bytes (51.2 KB). This is approximately 0.009% of the 700 MB system memory ceiling and small relative to the existing Ikigai data structures (transition_counts can reach 39+ pairs × small dicts; cluster_transition_counts up to 256 entries; novelty_counts grows to ≈40 keys). The buffer is the dominant cost (82% of the layer's footprint); doubling the buffer to 256 slots would still keep total VSA memory under 200 KB. This means future packs can comfortably add additional buffers (multiple chain registers, longer sequence memory, hypothesis vectors, option signatures) without approaching memory pressure. The constraint on buffer size is therefore not memory but the cosine-similarity computational cost: a per-event scan of a 128-slot buffer at 400-dim is ≈51200 byte comparisons; a 256-slot buffer doubles that. At Pack 4's current per-tick cost (sim_1 + sim_5 + sim_20 + recovery test + chain composition + pattern reuse scan), the dominant cost is the 128-slot pattern-reuse scan, which is 128 cosines per tick. A 1500-tick simulation completes in approximately 35 seconds with all VSA layers active, of which roughly 4-5 seconds is attributable to VSA per-tick cost (≈3 ms per tick). This is well within budget.

- **Zero behavioral drift is enforced by structural separation, not by parameter calibration.** Every Day 40 hook reads from existing IkigaiContext fields (`_ct_curr_cid`, `last_action_type`, `vsa_event`) but writes only to VSA-namespaced fields (`vsa_buffer`, `vsa_recover_*_sim`, `vsa_chain_event`). No VSA field is referenced by BG scoring, action selection, top_confidence assignment, routing logic, or any K-modulated variable. This is checked structurally at code review time (grep for vsa_ in non-state.py files would find only the controlled hook in ikigai.py and the experiment harnesses) and verified empirically by the bit-identical V1 tests in every pack. The pattern is the same as the always-active observation layers in Days 38-39 (transition_counts, cluster_transition_counts, ce/cpc accumulators): all observe state without modifying it, and the absence of a write path is the guarantee of zero drift. Day 41 will be the first day where a VSA-derived signal influences another part of the system, and the integration will be observation-only on the receiving side as well (a signal added to a running average, not used as a gate or threshold).

## 6. Failure Modes / Constraints

**Pack 2 buffer initialization timing for memory-stability test.** The Pack 2 V5 validation initially captured `len(_IKG.vsa_items)` at every 100th tick, including tick 0. On tick 0, the post-action region has not yet executed any cluster-transition block (no cluster has been assigned yet), so the lazy init path inside the VSA hook never fires, and `vsa_items` remains empty. The first non-empty sample appears at tick 100 (when the Pack 1 cluster rebuild completes and the next tick's transition logging triggers VSA encoding). This produced a sample series [0, 27, 27, 27, ...] and failed the "constant count" assertion for the first sample. The fix was to add eager initialization in the experiment harness alongside the flag setup: when `enable_vsa = True`, call `_IKG.vsa_init_items()` immediately at construction time rather than waiting for the lazy path. This is appropriate because the items are deterministic (seeded) and idempotent; eager init does not change any behavior. The lesson is that lazy initialization in observation-only layers can produce unexpected sampling patterns at simulation start; harnesses that sample state from tick 0 should either skip the warmup ticks or pre-initialize the structures they intend to measure.

**Pack 3 prev_cid snapshot ordering.** The Pack 2 hook (added in initial Day 40 work) ran AFTER the existing line `_IKG._cluster_prev_id = _ct_curr_cid` in the post-action region. This meant that by the time the VSA hook executed, `_IKG._cluster_prev_id` already held the current tick's cluster ID, not the previous tick's. Pack 3's classification logic, which compares prev_cid to curr_cid to detect cluster changes, would always see prev_cid == curr_cid and report zero cluster-change spikes -- a structurally invalid measurement. The fix was to introduce a local variable `_vsa_prev_cid = _IKG._cluster_prev_id` BEFORE the assignment line, and to pass that snapshot to `vsa_record_event` as the prev_cid argument. The same pattern repeats in Pack 4 for the chain composition: `_vsa_prev_for_chain = _IKG.vsa_event` must be captured BEFORE `vsa_record_event` mutates `vsa_event` to point at the new event. Both snapshots are zero-cost when the flag is False (the assignment to a local variable that is never read has no observable effect). The general principle: any signal that depends on a "before" value must be captured before the field is overwritten, even if the intervening code looks unrelated.

**Pack 3 spike threshold collision with Pack 2 baseline.** Pack 3 raised the spike threshold from Pack 2's 0.3 to 0.5 to align with the semantic interpretation that a spike represents a meaningful structural change rather than a small fluctuation. This change retroactively affects Pack 2's V4.2 assertion (which had counted spikes at the 0.3 threshold). To avoid breaking back-compatibility with previously-validated harnesses, both `vsa_spike_count` (canonical Pack 3 name, threshold 0.5) and `vsa_sim_spikes` (legacy alias, same value) are incremented together; the threshold field `vsa_spike_threshold` is shared between Pack 2 and Pack 3 but is now 0.5. Re-running the Pack 2 V4 assertion with the new threshold still passes -- the only change is that the absolute spike count is higher (more events fall below 0.5 than below 0.3), but the structural property "spike count > 0" remains. The lesson is that field renames in observation-only layers should keep the old name as an alias rather than introducing an ABI break; threshold parameter changes within an observation layer are acceptable when no external code depends on the absolute count.

**2-vector bundle threshold edge case.** The vsa_bundle implementation uses strict-majority threshold (sum > N/2) which for N = 2 reduces to sum > 1.0, equivalent to bitwise AND. This was an intentional choice (consistent with the strict majority for odd N) but produces an unexpected interpretation for the 2-vector chain composition: chain coherence in stable regimes is 1.0 (chain = AND of identical events = the same event), but in transition regimes chain coherence drops to ≈0.5 (the AND of two distinct random vectors has 25% bit density, far below the original vectors' 50%). The Pack 4 V4.2 assertion (mean chain coherence > 0.6) passes only because the 86%/14% stable/transition mix produces a weighted mean of 0.93, well above the 0.6 threshold. Were the regime to flip (more transitions than stable), the mean coherence could fall below 0.6. The correct interpretation of the 2-vector chain in this implementation is "similarity to the AND-projection of both events," which is approximately the similarity to whichever event is more bit-sparse (since AND biases toward sparse vectors). For Day 41 work involving 3-vector or longer bundles, this edge case dissolves -- 3-vector majority is balanced and produces uniform ≈ 0.5 similarity to each component. The 2-vector edge is therefore acknowledged but not corrected: the chain primitive is intended for short-term sequence reasoning (3+ events) where the mathematical balance is restored.

**Item-vocabulary scope is currently fixed and predetermined.** The 27 items (CID_0 through CID_15, six action types, five Pack 1 base concepts) cover the maximum possible cluster count (16, set by Day 39's max_clusters parameter) and the full action vocabulary (six types covering both raw BG and exec_diversity-mapped variants). However, the items are pre-allocated regardless of which clusters are actually formed during a given run; in a typical 1500-tick run, only 6-10 clusters are populated, leaving 6-10 CID_i vectors as unused dead weight. This wastes ≈4 KB of item memory in the average case. More importantly, the item vocabulary cannot grow at runtime: if a new action type is introduced (say, by future packs), it would have no corresponding ACT_<name> vector and `vsa_encode_event` would return None, silently dropping the event. The current design accepts this limitation as a Day 40 simplification; Day 41+ will need either (1) lazy vocabulary growth on first encounter (with a hard cap) or (2) explicit vocabulary expansion when new action types are introduced. The Pack 1 idempotence guard in `vsa_init_items` (return immediately if items are non-empty) prevents accidental re-initialization but also prevents intentional vocabulary growth; this guard would need to be relaxed or replaced with per-name conditional initialization for runtime expansion.

## 7. Architectural Insights

**The representation layer is structurally inert and intentionally so.** Day 40's four packs add 25 IkigaiContext fields, four core methods, three property accessors, and a small post-action hook in ikigai.py. The fields hold per-tick events, sums, counts, and one register of working memory. None of these fields are read by any code outside the experiment harnesses and the controlled VSA hook. The hook itself reads existing fields (`_ct_curr_cid`, `last_action_type`) but writes only to VSA-namespaced fields. This is the Day 38-39 observation-before-control discipline applied at the representation level: the VSA layer measures what is happening (it encodes every tick's behavior into a vector), tests its own properties (recovery, chain coherence), and accumulates aggregate statistics, but does not influence any decision the system makes. The discipline is not aesthetic. The architectural risk in any representation system is that a flawed representation, once wired into control, biases all downstream decisions toward its flaws. By proving the representation works in isolation across 71 assertions before connecting it to any control loop, Day 40 ensures that Day 41+ integrations can rely on the representation layer's basic correctness.

**Vector Symbolic Architecture compresses Ikigai's existing state space without any new training.** The Day 39 cognitive scaffold operates over coarse state keys (≈13 distinct values), cluster IDs (6-10 active), action types (3-6 depending on exec_diversity remap), and various scalar signals (energy, PE, conflict). The state space is therefore small in cardinality but high in conditional complexity: the system reasons about transitions among these states, conditional probabilities given prior states, and confidence in those probabilities. VSA replaces this small-cardinality, conditionally-dense representation with a fixed-cardinality, structurally-uniform representation: every (cid, action) pair maps to a 400-dim binary vector via XOR composition, and the resulting vectors have a clean cosine-similarity structure that respects the underlying cluster geometry. There is no training step: the item vectors are random and fixed; the bind/bundle/similarity operations are deterministic; the per-tick encoding is a single XOR of two pre-allocated vectors. The system's existing learning machinery (transition counts, cluster transition counts, novelty counts, curiosity values) is unaffected by Day 40 and continues to learn at its own time scale. VSA is not a learning system; it is a static algebra over the system's existing state, providing a substrate on which future learning systems can operate. The distinction between substrate and learner is important: future packs may add learning over VSA representations (e.g., associating bundled chains with reward outcomes), but Day 40 itself adds zero learning capacity.

**Per-tick computational cost is predictable and bounded.** The VSA hook runs in O(D) for binding (single XOR over 400 bytes), O(D × K) for bundling K vectors, O(D) for cosine similarity, O(D × W) for sim_5 and sim_20 windowed similarity (where W is the window size, 5 or 20), and O(D × B) for pattern reuse (where B is the buffer size, 128). The dominant cost is the pattern reuse scan: 128 cosines per tick at 400 bytes each = 51,200 byte comparisons. At a Python-loop cost of roughly 10 ns per byte comparison, this is ≈0.5 ms per tick of overhead, plus similar costs for sim_5/sim_20 and the per-tick recovery test. Empirically, total VSA per-tick cost is ≈3 ms in the current implementation (1500 ticks × 3 ms = 4.5 s of overhead in a 35 s total run). This is acceptable but not negligible; a future optimization opportunity is to use NumPy or bit-packed integers for the cosine computation, which would reduce the cost by 10-50x. The optimization is not urgent for Day 41 (which adds no per-tick VSA work beyond what Day 40 already does), but will become relevant when option signatures (Day 43+) require per-tick scans against a larger option library.

**Compositionality is the property that distinguishes representation from logging.** The Day 40 buffer could be implemented as a list of (cid, action) tuples rather than as a list of bound hypervectors -- the same multi-scale similarity tests could be run by checking tuple equality rather than cosine similarity. The architectural difference is that bound hypervectors compose: chain = bundle(E_t, E_{t-1}) is itself a vector of the same shape as E_t and E_{t-1}, and can be bound, bundled, or similarity-compared with any other vector in the system. A list of tuples cannot. This compositional property is what makes VSA the right substrate for planning, hypothesis generation, and option signatures: a hypothesis "I expect to be in cluster X after performing action Y" is naturally a single vector bind(CID_X, ACT_Y); a planned sequence "do A then B then C" is naturally a bundle of three bound events; a procedural memory "this option was selected at events matching pattern P" is naturally a vector similarity test. None of these constructions are possible with a tuple-list representation; all are immediate with the VSA substrate. The Day 40 packs do not yet exercise these constructions, but they prove the substrate supports them: bind is exact, bundle preserves components above 0.6 cosine, and similarity respects cluster geometry.

## 9. Next Direction (NOT IMPLEMENTED)

Day 40 establishes a structurally-correct, semantically-grounded, computationally-capable representation layer. The next architectural step is to introduce the first observation-only diagnostic coupling between the VSA layer and an existing system signal. The chosen target is the prediction error stream: each tick's bound event can be compared against a "predicted next event" computed from the existing transition_probs (state-level argmax next state) and cluster_transition_probs (cluster-level argmax next cluster) -- both of which are already maintained as observation-only signals. The prediction-vs-actual cosine similarity in VSA space is then a parallel measurement to the existing transition_error scalar, with the difference that the VSA-based measurement has graded similarity (cosine in [-1, +1]) rather than the binary hit-or-miss of the lookup-based transition_error. Day 41 Pack 1 will introduce this signal as a pure observation-only scalar `vsa_prediction_similarity`, accumulated alongside the existing TE/PC aggregates without entering any scoring path. The goal is to verify that the VSA-based prediction signal correlates with the existing transition_error (high TE should correspond to low VSA cosine) before any consideration of using either signal for control.

The path beyond Day 41 follows the May 1 strategic plan: VSA → Options Framework (Day 42-44) introduces the (I, π, β) tuple where each option's signature is a bundled chain of bound events, allowing the system to identify "I have seen this kind of situation before" by cosine similarity to stored option signatures; VSA → Active Inference (Day 45-48) uses the bound-and-bundled events as the substrate for expected-free-energy computation, where pragmatic value is the energy/PE prediction from existing world-model machinery and epistemic value is computed from the VSA signal's reducible uncertainty. Both directions inherit Day 40's representation layer as a fixed substrate; neither modifies the binding, bundling, or similarity primitives. The Day 41 prediction-correlation measurement is the smallest possible step that connects VSA to an existing system signal without introducing any control coupling, and it is the necessary precursor to the larger integrations.

A second direction is sequence memory: extending the chain composition from 2 vectors to longer sequences (3-7 events) using the strict-majority bundling, which produces balanced sim ≈ 0.5 to each component for odd-length bundles. Sequence vectors of this kind are the natural representation for short-term episodic memory and for option-trace identification. The implementation would add a sequence buffer (separate from the per-event circular buffer) accumulating events between regime transitions, and a periodic sequence-summary computation that bundles the buffer into a single hypervector when a regime transition fires (a Pack 6 conflict event, for instance). This is a Day 41-42 concern; the substrate for it is fully present today.

A third direction, lower-priority, is item vocabulary expansion to handle runtime-introduced action types. The current 27-item vocabulary covers all action types observed in Days 36-39 but cannot grow. The expansion would relax the `vsa_init_items` idempotence guard to allow per-name conditional initialization, with a hard cap (e.g., 64 items maximum) to bound memory. The expansion is not blocking for any specific Day 41+ work but is a known limitation worth addressing before vocabulary needs change.
