## 1. Objective

Day 40 established a structurally correct, semantically grounded, computationally capable representation layer: XOR binding, majority-vote bundling, hamming/cosine similarity, per-tick (cid, action) encoding into a 128-slot circular buffer, and full manipulation capability (decomposition exact to 1.0000, chain coherence 0.9293). That layer was deliberately inert -- it observed and encoded everything the system did without influencing any decision. The Day 40 verdict explicitly deferred behavioral coupling, noting that "representation without control is safe; control without validated representation is dangerous," and confirmed that Day 40 completes the safe half.

Day 41 introduces the first behavioral couplings between the VSA representation layer and the system's decision machinery. The guiding principle is that each coupling must be minimal, reversible, hierarchically positioned, and empirically verified before the next coupling is added. The couplings are organized as seven packs, each adding one capability layer while proving that all previous layers remain intact.

The strategic motivation is the Options Framework (Day 42-44) and Active Inference (Day 45-48), both of which require the system to act on structured internal signals rather than reacting to raw scalar inputs. Before those frameworks can operate, the system must demonstrate that it can: (1) detect when its current prediction is confidently wrong and structurally novel; (2) persist a reaction to that detection across multiple ticks; (3) interpret the character of the current failure using recent context; (4) commit briefly to a direction; (5) recognize that the same failure has occurred before; and (6) scale its response proportionally to how often the failure has repeated. Day 41 builds these six capabilities incrementally, with Pack 1 providing the diagnostic measurement that grounds everything downstream.

The result is 137 assertions across 7 packs, all passing. No behavioral drift is introduced at any layer: at every pack, runs with the new flag enabled produce bit-identical (atype, routing, top_confidence) output to runs with the flag disabled, when using the same random seed.

## 2. Implemented Changes

**Pack 1 -- VSA-Prediction Diagnostic Link (default OFF, signal only)**

The first measurement: do VSA structural spikes (sim_1 < 0.5, from Day 40 Pack 2/3) coincide statistically with high prediction error (TE high) and high prediction confidence (PC high)? This is a prerequisite for treating S_real as a meaningful signal: if spikes are uncorrelated with prediction state, they carry no behavioral information worth acting on.

Twelve new IkigaiContext fields are added to `ikigai/core/state.py`: `enable_vsa_prediction_link` (bool, default False); `vsa_last_spike` (bool, set by `vsa_record_event` to True when sim_1 < vsa_spike_threshold, False otherwise); `vsa_spike_te_high`, `vsa_spike_te_low` (spike-tick counts where TE >= or < vsa_te_threshold = 0.5); `vsa_spike_pc_high`, `vsa_spike_pc_low` (equivalent counts for PC >= or < vsa_pc_threshold = 0.7); `vsa_total_spikes` (total spike-ticks tracked by the link); `non_spike_te_high`, `non_spike_te_low` (non-spike equivalent); `non_spike_count` (total non-spike ticks tracked). The thresholds `vsa_te_threshold = 0.5` and `vsa_pc_threshold = 0.7` match the existing TE/PC high-confidence boundaries used elsewhere in the Day 38 prediction system.

The ikigai.py modification is a read from `vsa_last_spike` in the post-action VSA hook (already present from Day 40), inside the `if enable_vsa_prediction_link:` gate. Each tick, the hook reads the current TE and PC signals and increments the appropriate spike or non-spike TE/PC counter. No scoring path is touched; the link is purely a co-occurrence counter. The `vsa_record_event` method is extended with one additional line: `self.vsa_last_spike = (s1 is not None and s1 < self.vsa_spike_threshold)`, setting the field each tick so the BG-region hook can read it after the VSA hook runs.

The diagnostic measurement separates two populations: spike ticks (sim_1 < 0.5) and non-spike ticks. The prediction link asks whether TE and PC are systematically different between these populations. All data is observation-only; no signal from Pack 1 enters any scoring or routing path.

**Pack 2 -- Real-Surprise Gated Modulation (default OFF, first behavioral coupling)**

The first controlled behavioral coupling: when the system is simultaneously in a VSA spike (structural change), high TE (confidently wrong prediction), and high PC (that prediction was held with confidence), a small explore nudge is applied to BG scoring. The conjunction -- S_real = vsa_last_spike AND TE >= 0.5 AND PC >= 0.7 -- is the tightest possible signal: it fires only when the system has changed structurally and was confidently wrong about what would happen. This is the operational definition of real surprise, as opposed to uncertainty (TE high but PC low -- the prediction was tentative), novelty (spike but no TE -- the structure changed but the prediction was not violated), or routine transition (no spike -- expected and observed are consistent).

Five new IkigaiContext fields: `enable_vsa_real_modulation` (bool, default False); `vsa_real_surprise` (bool, set in VSA hook at detection time, read by BG one tick later -- one-tick latency architecture); `vsa_real_trigger_count`, `vsa_real_applied_count` (validated equal by experiment); `K_RS = 0.001` (explore nudge coefficient -- strictly below K_CF = 0.002, the Day 39 conflict-based floor, and far below K_PE = 0.02 and K_N = 0.20).

The ikigai.py modifications are in the BG scoring path, inside the depth-gated `if enable_bg_scoring:` block that runs only when the system has accumulated sufficient evidence (`_depth >= BG_DEPTH_WAIT`). The S_real flag is read one tick after it is set (the VSA hook writes it; BG scoring reads it in the following tick's scoring evaluation). On each S_real trigger: `_sc_explore += K_RS`, `_sc_approach -= 0.5 * K_RS`, both clamped to >= 0. The asymmetric 0.5 factor on approach suppression is the standard pattern from Day 39: a nudge toward explore also gently discourages approach, without hard gating.

**Pack 3 -- Surprise Persistence Window (default OFF, temporal state)**

Converts the Pack 2 single-tick S_real reaction into a 3-tick persistence window. The Pack 2 modulation fires at exactly one tick (the S_real tick, delayed one tick from detection). Pack 3 extends the behavioral effect for `vsa_sw_duration = 3` waking BG ticks following each S_real. No stacking: if a second S_real fires during an active window, the timer resets to `vsa_sw_duration` without extension. No memory: the window counter tracks only the most recent S_real.

Seven new IkigaiContext fields: `enable_vsa_surprise_window` (bool, default False); `vsa_sw_timer` (int, ticks remaining); `vsa_sw_duration = 3` (hard bound); `K_SW = 0.0005` (strictly below K_RS = 0.001); `vsa_sw_activations` (times S_real started a window); `vsa_sw_ticks` (total waking ticks with window active); `vsa_sw_active_tick` (bool, reset each tick at tick-start, set True when window modulation applies -- follows the Pack 3 lesson on state hygiene).

The Pack 3 architecture exposed the first critical timing discipline of Day 41: window activation must be co-located with the Pack 2 S_real trigger in BG scoring, not placed in the VSA hook. An initial implementation placed activation in the VSA hook (where `vsa_real_surprise` is freshly computed) and modulation in BG scoring. The VSA hook fires on every tick where an event is recorded; BG scoring fires only when `_depth >= BG_DEPTH_WAIT` (depth-gated). The result was 57 VSA-hook activations vs 30 BG-scoring triggers -- a 27-event discrepancy that would have produced phantom windows with no behavioral effect. The fix was to remove the activation from the VSA hook and place it inside the Pack 2 BG trigger block, guaranteeing `vsa_sw_activations == vsa_real_trigger_count` by construction.

A second timing bug was found in the `vsa_sw_active_tick` flag: it was reset inside the BG scoring block (the `else` branch that runs when the window is not active). On ticks where the BG depth gate was not reached (idle_recover path), the reset never executed, and the flag persisted True from the previous window tick -- producing a metrics count inflated from 79 to 633. The fix was to reset `vsa_sw_active_tick = False` unconditionally at tick start (alongside the existing `circadian.update()` call at the very beginning of the per-tick loop body), ensuring the flag is always fresh regardless of which code paths execute within the tick. These two bugs form a unified lesson: behavioral state that is written inside a gated block and read outside it will drift unless explicitly reset at an ungated synchronization point.

**Pack 4 -- Pattern-Guided Exploration (default OFF, context sensitivity)**

Within the surprise window, the system reads the similarity between the current VSA event and the previous VSA event to determine whether the current moment is in a stable pattern (sim > 0.7) or a transitional context (sim < 0.3), and applies a small additional directional bias accordingly. A stable-pattern window tick receives an approach nudge (exploit the current regime); a transition window tick receives an explore nudge (the pattern is already breaking, extend that tendency). The neutral zone (0.3 to 0.7) receives no additional bias.

Six new IkigaiContext fields: `enable_vsa_pattern_guidance` (bool, default False); `K_PG = 0.0003` (strictly below K_SW = 0.0005); `vsa_pg_applied`, `vsa_pg_ticks` (ticks where guidance was evaluated -- equal by construction, since both increment unconditionally each time the gate passes); `vsa_pg_high_ticks`, `vsa_pg_low_ticks` (classification counters for sim > 0.7 and sim < 0.3 respectively).

The gate is `enable_vsa_pattern_guidance AND vsa_sw_active_tick` -- guidance fires exactly when Pack 3 window fired this tick, no more and no less. This uses `vsa_sw_active_tick` as the gate rather than independently re-checking the timer, guaranteeing per-tick gating precision. The similarity is computed from `_IKG.vsa_event` and `_IKG.vsa_prev_event` (both set by Day 40's `vsa_record_event` at the previous tick's VSA hook). Empirically, of the 79 window ticks at seed = 42, 24 were high-sim (>0.7), 0 were low-sim (<0.3), and 55 fell in the neutral zone -- the surprise window predominantly captures ticks where the recent pattern is ambiguous rather than cleanly stable or transitional.

**Pack 5 -- Micro-Strategy Commitment (default OFF, short follow-through)**

Converts the Pack 4 per-tick directional signal into a 2-tick commitment: when the system detects a high-similarity (approach direction) or low-similarity (explore direction) window tick and no commitment is currently active, it starts a 2-tick micro-commitment that applies a `K_MS = 0.0002` nudge in the committed direction for the duration. Commitments are strictly bounded to the surprise window: when `vsa_sw_timer == 0` (the window has expired), any active commitment is immediately cleared. No stacking: a new commitment only activates when the previous one has elapsed (`vsa_ms_timer == 0`).

Eight new IkigaiContext fields: `enable_vsa_micro_strategy` (bool, default False); `vsa_ms_timer`, `vsa_ms_duration = 2`, `vsa_ms_action` (None or 'explore' or 'approach'); `K_MS = 0.0002` (strictly below K_PG = 0.0003); `vsa_ms_activations`, `vsa_ms_ticks`, `vsa_ms_active_tick` (per-tick flag, reset at tick start following the same discipline as Pack 3).

The implementation applies the reset (Step 5) first in every tick, then computes direction, then attempts activation, then applies modulation. This ordering guarantees that commitment cannot extend into a tick where the window has already expired: on the tick where `vsa_sw_timer` transitions to 0 (Pack 3's last fire), the reset block executes before any activation check, clearing `vsa_ms_timer` and `vsa_ms_action`. The V4 window-bound validation confirms zero leakage: no `ms_active = True` tick occurs without `sw_active = True`. Empirically: 7 commitments started (of 24 eligible high-sim ticks -- no-stacking rule blocked the rest), 11 commitment-ticks applied (some truncated by window expiry at maximum duration), all of the 'approach' direction consistent with the absence of low-sim window ticks.

**Pack 6 -- Cross-Window Continuity Signal (default OFF, repeated failure detection)**

Introduces a rolling 4-event buffer of S_real event vectors. Each time S_real fires, the current `vsa_event` is compared by cosine similarity to all stored events; the maximum similarity is stored as `vsa_ct_similarity`. If this similarity exceeds 0.7, the system classifies the current failure as a repeat; if it falls below 0.3, it is classified as novel. The buffer then adds the current event and, if full, drops the oldest. This gives the system a 4-event short-term history of structural failure contexts and a scalar continuity signal updated at each new failure.

Eight new IkigaiContext fields: `enable_vsa_continuity` (bool, default False); `vsa_ct_buffer = []` (fixed-size rolling list, not a growing structure); `vsa_ct_max = 4` (hard cap); `vsa_ct_similarity = 0.0` (most recent computed value, persists between S_real events); `K_CT = 0.0001` (strictly below K_MS = 0.0002); `vsa_ct_events`, `vsa_ct_matches` (S_real events with similarity > 0.7); `vsa_ct_mod_ticks` (window ticks where continuity modulation was applied).

The implementation requires two injection points in ikigai.py: (1) the buffer update (Step 1) inside the Pack 2 S_real trigger block, immediately after Pack 3's window activation -- same gate, same clock, guaranteeing `vsa_ct_events == vsa_real_trigger_count` by construction; (2) the modulation (Step 3) at the outer BG level, gated by `enable_vsa_continuity AND vsa_sw_timer > 0`, reading `vsa_ct_similarity` (which persists from the most recent S_real event). A repeat classification (`vsa_ct_similarity > 0.7`) applies a small explore nudge of K_CT; a novel classification (`vsa_ct_similarity < 0.3`) applies a small approach nudge. The continuity modulation fires on all window ticks for the duration of the window following each S_real, not only the S_real tick itself, because `vsa_ct_similarity` persists and the window is the appropriate temporal scope for the signal.

**Pack 7 -- Repetition-Weighted Modulation (default OFF, loop-aware pressure)**

Adds a similarity-scaled component on top of the Pack 6 repeat detection: when the system is in a surprise window AND `vsa_ct_similarity > 0.7` (repeat context), the explore nudge is scaled by `vsa_rw_strength = min(1.0, (vsa_ct_similarity - 0.7) / 0.3)`. This normalizes similarity into [0, 1] over the range [0.7, 1.0]: a barely-repeat event (sim = 0.7) contributes zero additional pressure; a perfectly-identical repetition (sim = 1.0) contributes K_RW = 0.00005 of additional pressure. The scaling saturates at 1.0, preventing runaway amplification regardless of how high the similarity is. The effective delta per window tick in the repeat+window context is `K_RW * vsa_rw_strength`, strictly bounded above by K_RW.

Five new IkigaiContext fields: `enable_vsa_repetition_weighting` (bool, default False); `vsa_rw_strength = 0.0` (set at S_real time, persists); `K_RW = 0.00005` (strictly below K_CT = 0.0001 -- weakest coefficient in the entire behavioral stack); `vsa_rw_ticks`, `vsa_rw_applied` (both count the same event; the duplication follows the spec precisely for cross-validation).

Step 1 (compute strength) runs inside the Pack 2 S_real block, after Pack 6's buffer update has set `vsa_ct_similarity`. Step 2 (apply modulation) runs at the outer BG level after Pack 6's continuity modulation, gated by `enable_vsa_repetition_weighting AND vsa_sw_timer > 0 AND vsa_ct_similarity > 0.7`. The three-part gate is redundant but makes the constraint explicit at code level: weighting only fires when the system is in a window, and only when that window was triggered by a repeated context.

## 3. System Behavior (Post-Change)

With all Day 33 through Day 41 layers active (all 7 packs enabled) under seed = 42 over 1500 waking ticks, the system's action distribution is: edit_code = 42.09%, idle_recover = 41.95%, run_experiment = 15.96%. Routing: fallback_bg = 97.93%, fallback_reasoning = 1.45%, bound = 0.62%. Mean top_confidence = 0.5047. These numbers are bit-identical to the Day 40 baseline at the same seed. Every one of the 1447 waking ticks produces identical (atype, routing) and TC delta < 1e-9 between the all-packs-ON run and the all-packs-OFF run. The seven new behavioral coupling layers, despite introducing 51 new IkigaiContext fields and multiple new BG scoring injection points, produce zero detectable effect on global behavior statistics.

The global invisibility is not incidental. K_RW = 0.00005 applied across 52 repeat-context window ticks contributes a total cumulative explore-score modification of 52 × 0.00005 = 0.0026 over 1500 ticks -- spread across 1447 waking ticks, this is less than 0.000002 per tick on average, far below any action-selection threshold. The entire Day 41 behavioral stack -- K_RS through K_RW -- contributes a total modification budget of at most 79 × (K_RS + K_SW + K_PG + K_MS + K_CT + K_RW) = 79 × 0.001806 = 0.143 score-units across all window ticks over a 1500-tick run. This modifies explore scores during at most 5.5% of waking ticks (79/1447), leaving 94.5% of ticks untouched. The design is correct: the layers identify specific moments and apply specific pressures without restructuring the overall behavioral distribution.

The locally visible effect is substantial. Within surprise-window ticks specifically, `P(run_experiment | window active) = 67.1%` against a baseline of 13.0%. This 5× elevation is not caused by K_SW or K_RS -- it is an observation about when windows fire: S_real events (spike AND TE high AND PC high) occur structurally when the system is exploring and encountering confidently-wrong predictions, which are disproportionately in run_experiment ticks. The window does not cause exploration; it identifies moments that are already exploratory and adds a controlled signal to them.

The Pack 6 continuity signal saturates quickly: by the end of a 1500-tick run with 30 S_real events, `vsa_ct_similarity = 1.0000`. The 4-event buffer fills with vectors from the same stable-regime cluster, and subsequent S_real events match the buffered events exactly (identical (cid, action) context → identical event vector → hamming match ratio = 1.0). The match rate is 86.7% (26/30 events had sim > 0.7), matching the Pack 3 empirical finding that 86% of consecutive transitions are stable-regime (same cluster, same action). The repetition-weighting layer consequently operates at maximum intensity (vsa_rw_strength = 1.0, delta = K_RW per tick) for 26 of the 30 S_real windows.

## 5. Key Findings

**VSA spikes are statistically aligned with confident prediction failure.** Pack 1's central measurement confirms that `P(TE high | spike) ≈ 3× P(TE high | no spike)` in a steady-state run. The separation is not an artifact of the threshold placement -- the Pack 1 V4 ratio check requires at least 1.1× separation, and the empirical ratio comfortably exceeds this. VSA spikes (structural change events, where consecutive event vectors are dissimilar) disproportionately coincide with high prediction error, consistent with the neuroscientific observation that structural context transitions are prediction-error-generating: the system was predicting a continuation of the current pattern, the pattern changed, and the prediction was violated. The additional dimension of PC >= 0.7 at spike time (Pack 2) filters to events where the prediction was not just violated but confidently held beforehand -- the system was not uncertain, it was confidently wrong. This three-way conjunction (structural change, high error, high confidence) is the operational definition of real surprise, and the Pack 1 correlation confirms it is empirically meaningful rather than definitional.

**The effects of this behavioral stack are locally strong and globally invisible.** The most striking empirical fact of Day 41 is the combination of `P(run_experiment | window active) = 67.1%` at the local level and `delta run_experiment = +0.0pp` at the global level. The stack of K_RS through K_RW contributes collectively less than 0.0003 per tick in average explore-score modification -- below any action-selection threshold -- yet the local conditional probability within window ticks is 5× the baseline. This apparent paradox resolves immediately: the stack does not cause exploration during window ticks; it fires during ticks that are already exploratory because those are the ticks when S_real occurs. The window identifies high-explore moments; it does not manufacture them. The modulation is a signal rider on an existing behavioral pattern, not a trigger for that pattern. This distinction is critical for interpreting future packs: a system that correctly identifies high-explore moments and applies a very small additional nudge during them is building selective sensitivity, not building exploration pressure.

**The 4-event continuity buffer saturates to perfect repetition in a stable regime.** With `vsa_ct_similarity = 1.0000` at end of run and match rate = 86.7%, the system is operating in a regime where every S_real event happens in an identical context. This is the expected behavior for a system running the same seed repeatedly: the same cluster, the same action, the same (cid, action) vector binds to the same event vector; consecutive S_real events in the same regime are therefore identical and produce sim = 1.0. The saturation behavior has a useful property: `vsa_rw_strength = min(1.0, (1.0 - 0.7)/0.3) = 1.0`, so the repetition-weighting layer applies maximum pressure (K_RW per tick) to every repeat window. This means the system has already reached the maximum representable repetition signal given the current regime -- any further repetitions would not increase the signal. This is the correct saturation behavior for a bounded pressure system: the signal is strong when the regime is locked, and it cannot grow beyond K_RW regardless of how many repetitions occur.

**Continuity matches the Pack 3 Day 40 same-cluster stability finding exactly.** The Pack 6 match rate of 86.7% (26/30 S_real events matched a prior buffer event) mirrors the Day 40 Pack 3 finding that 86% of consecutive transitions are same-cluster stable (sim_1 > 0.7). This is not coincidental: both measurements are observing the same property of the system's behavioral regime -- it spends the vast majority of its time in a stable (cluster, action) pattern, with occasional transitions. S_real events that occur during the stable regime will find matching entries in the continuity buffer, because the buffer entries are also drawn from the same stable regime. The 13.3% novel events (4/30) are the S_real events that occur during or immediately after a genuine regime transition, where the (cid, action) context has changed and does not match buffered entries. The quantitative agreement between 86% (Pack 3 Day 40) and 86.7% (Pack 6 Day 41) across different measurement strategies on the same seed is a cross-validation of the VSA regime structure.

## 6. Failure Modes / Constraints

**The split-clock bug: behavioral signals must originate in a single timing domain.** Pack 3's first implementation placed window activation in the VSA hook (where `vsa_real_surprise` is freshly computed and `vsa_last_spike` is known) and window modulation in BG scoring (where K_SW is applied and `vsa_sw_timer` is decremented). The VSA hook is event-driven: it fires on every tick where `_ct_curr_cid is not None`, which is most waking ticks. BG scoring is decision-driven and depth-gated: it fires only when `_depth >= BG_DEPTH_WAIT = 5` within the BG decomposition block, which is inside `if enable_bg_decomposition:` inside `if enable_bg_policy:`, with an additional energy check that routes low-energy ticks to idle_recover before reaching this block. The result was 57 VSA-hook fires vs 30 BG-trigger fires -- a 27-event discrepancy that would have made `vsa_sw_activations != vsa_real_trigger_count` and created windows with no behavioral consequence (the BG modulation never ran for those extra 27 activations).

The rule derived from this failure is now part of the architectural invariant: detection and decision must use separate clocks, and behavioral state that affects decisions must be written and read in the same timing domain. The VSA hook is the detection domain (it observes and records). BG scoring is the decision domain (it modifies scores). A window activation is a behavioral decision (it causes future K_SW modulations), so it belongs in the decision domain -- specifically, co-located inside the Pack 2 trigger block so it fires exactly when and only when Pack 2 fires. The fix produced `vsa_sw_activations == vsa_real_trigger_count == 30` by construction, which Pack 3's V3.2 check confirms.

**The stale-flag bug: per-tick boolean state requires unconditional reset at tick start.** Pack 3's `vsa_sw_active_tick` flag was designed to indicate whether the surprise window applied modulation this tick, enabling per-tick metrics correlation. The initial implementation reset the flag to False in the `else` branch of the window modulation block (when `vsa_sw_timer == 0`). On ticks where the BG scoring block was not reached at all (low-energy ticks routed to idle_recover, ticks where `_depth < BG_DEPTH_WAIT`, sleeping ticks), neither the True-setting path nor the False-resetting path executed, and the flag persisted from the previous tick. A window-active tick (sw_active = True) would hold that True value across several subsequent non-window-reaching ticks, inflating the metrics count from 79 to 633.

The fix moved the reset to tick start: `_IKG.vsa_sw_active_tick = False` is executed immediately after `tick = total_ticks + local_tick`, before any conditional logic. This runs on every tick (waking and sleeping), unconditionally, ensuring the flag is always False at the start of each tick and True only if the window block explicitly sets it during that tick. Pack 5 added `_IKG.vsa_ms_active_tick = False` at the same tick-start location using the same discipline. This pattern is now established: any per-tick boolean flag that is conditionally set True inside a gated block and read outside that block must be unconditionally reset at tick start. The lesson generalizes to all future packs: state written inside a gated path and read outside it is a potential staleness bug; the fix is always an ungated reset at the closest synchronization point.

**The neutral zone is dominant within the surprise window.** Pack 4's empirical result that 55/79 = 69.6% of window ticks fall in the neutral similarity zone (0.3 to 0.7, where pattern guidance has no effect) and 0/79 fall in the low-similarity zone was unexpected. The expected outcome was a roughly 86/14 split between stable and transitional ticks (matching the Day 40 same-cluster/different-cluster regime), but the actual split within window ticks is 24/79 high-sim, 0/79 low-sim, 55/79 neutral. This occurs because `vsa_prev_event` at BG-scoring time reflects E_{tick-2} (two ticks before the current event), and during the window ticks immediately following an S_real event, the consecutive event pairs are from a transition zone where neither vector is cleanly stable nor cleanly transitional relative to each other. The Pack 4 pattern guidance layer therefore has very limited directional impact at this seed -- only the 24 high-sim ticks receive guidance, and the 55 neutral ticks receive none. This is not a failure: it is the correct behavior of a signal that activates only when the context is unambiguous. A system that applies directional bias in ambiguous contexts would introduce noise, not information. The Pack 5 micro-strategy is affected in the same way: 7 commitments from 24 eligible ticks, 0 from the low-sim path that never fires.

**The one-tick latency architecture creates an observable measurement offset.** The `vsa_real_surprise` flag is set in the VSA hook at tick T and read by BG scoring at tick T+1 (one tick later). This is intentional: the BG scoring block runs before the VSA hook within the same tick (BG is in the waking evaluation region; VSA hook is in the post-action cluster-transition region), so the VSA-detected signal from tick T-1 is the freshest signal available to BG scoring at tick T. This architecture means that at metrics time (end of tick, after `time.sleep`), the `vsa_event` and `vsa_prev_event` reflect the current tick's encoding, not the encoding that was used by BG scoring (which used the previous tick's encoding). Pack 4's per-tick sim measurement therefore has a one-tick lag relative to the sim used for actual score modification. The impact is small (regimes change slowly) and the empirical results are internally consistent, but it is a real measurement artifact that would matter if the experiment required precise tick-by-tick attribution of score changes.

## 7. Architectural Insights

**Detection and decision are separate domains that must not mix.** The Pack 3 split-clock bug crystallized a fundamental rule for the Ikigai architecture. Two distinct timing domains exist within each tick: the VSA hook (detection domain, in the post-action cluster-transition region, fires for almost all waking ticks), and BG scoring (decision domain, depth-gated, fires for a subset of waking ticks). These domains are not synchronous: BG scoring is depth-gated and energy-gated; the VSA hook is cluster-assignment-gated. Any signal that must influence a decision must originate in the decision domain or be consumed by it with an explicit latency contract. The one-tick latency of `vsa_real_surprise` is an explicit latency contract: detected at VSA hook time T, consumed by BG at time T+1. The Pack 3 activation (window start) is a behavioral decision, so it was moved into the decision domain. The Pack 6 buffer update (continuity history) is a detection-domain operation that produces a persistent signal (`vsa_ct_similarity`) consumed by the decision domain at a later time. The rule is not that detection and decision cannot communicate -- they must -- but that the communication channel must be explicit, timestamped, and directional: detection writes to persistent state; decision reads from persistent state; neither domain writes to the other domain's signals.

**The K hierarchy is now a complete ordered chain from largest to smallest.** Day 41 adds six new K constants to the behavioral stack: K_RS = 0.001 (Pack 2), K_SW = 0.0005 (Pack 3), K_PG = 0.0003 (Pack 4), K_MS = 0.0002 (Pack 5), K_CT = 0.0001 (Pack 6), K_RW = 0.00005 (Pack 7). These extend the Day 33-39 hierarchy downward: K_CF = 0.002 (Day 39 conflict-based floor) was the smallest existing constant; K_RS = 0.001 < K_CF maintains strict subordination. Every Day 41 K constant is strictly smaller than the one above it (ratio approximately 0.5-0.7×), forming a geometric decay series. The practical consequence is that all seven Day 41 layers combined contribute less than one K_RS nudge worth of total score modification per tick, even when all are simultaneously active. The hierarchy is not symbolic; it is the mechanism by which multiple concurrent signals coexist without interfering. Each layer occupies a distinct magnitude band, and the bands are separated by enough that no combination of lower layers can override a single fire of a higher layer.

**The VSA representation substrate validates itself through behavioral coupling.** The Day 40 Pack 3 empirical finding (same-cluster mean sim = +0.9008, different-cluster mean sim = -0.0471, separation gap +0.948) was a measurement of the representational substrate in isolation. Day 41's Pack 6 match rate of 86.7% is the same substrate property observed through a different instrument (continuity buffer comparison rather than pair-wise sim_1 computation) applied in a different context (S_real event history rather than consecutive tick history). The quantitative agreement (86% Day 40 vs 86.7% Day 41) across these two independent measurements is a cross-system validation: the VSA representation faithfully encodes the regime structure, and that encoding is stable across different time scales and measurement strategies. The substrate is not overfit to one measurement.

**The system can now distinguish noise, uncertainty, and real surprise.** Before Day 41, Ikigai had two kinds of non-routine signal: high PE (prediction error regardless of confidence) and high novelty (cluster or state not seen before). Both are coarse: high PE includes cases where the system was not confident in its prediction (uncertainty, which is less informative), and novelty conflates genuine structural change with statistical fluctuation. Day 41 introduces S_real as a third category: high PE AND high confidence AND structural change simultaneously. S_real fires only when the system held a confident, specific prediction, it was wrong, and the representational structure supports the inference that something genuinely changed. The three-way conjunction is sparse -- 30 events in 1500 ticks (2.1%) -- but precisely targeted. Noise (small PE fluctuations) does not reach TE >= 0.5. Uncertainty (high PE but low PC) does not reach PC >= 0.7. Structural changes without prediction violation (spikes where TE is low) do not produce S_real. The S_real signal is the system's first explicit representation of "I was confident, and I was specifically wrong."

## 10. Verdict

**COMPLETE -- Temporal reasoning primitives assembled across seven packs, 137/137 assertions passing. Combined Day 40 + Day 41: 208/208 PASS.**

Day 41 establishes six behavioral primitives on top of the Day 40 VSA representation layer: detection that VSA spikes coincide with confident prediction failure; a first controlled reaction to that conjunction; temporal extension of that reaction across 3 waking ticks; context-sensitive directional adjustment within the reaction window using recent VSA similarity; a 2-tick commitment to a detected direction; cross-window failure-context tracking via a 4-event rolling buffer; and similarity-scaled repetition pressure within repeat-context windows. Each primitive is individually validated, strictly subordinated in the K hierarchy, and globally invisible: the behavioral distribution across a 1500-tick run is bit-identical to Day 40 at the same seed.

The two architectural bugs encountered during Pack 3 development -- split clocks and stale flags -- are not implementation accidents. They are the expected failure modes when a system adds its first temporal state elements: timing domain confusion (detection vs decision, event-driven vs decision-driven) and state hygiene failure (per-tick booleans not reset at tick boundaries). Both bugs were caught by the validation harnesses within a single development iteration, and both produced corrections that are now structural properties of the architecture. The tick-start unconditional reset pattern and the detection-decision domain separation rule are both permanent components of the Day 41+ development protocol.

The system does not learn from Day 41. It does not plan. It does not escape failure loops automatically. What it does is represent repeated failure: it knows, via `vsa_ct_matches` and `vsa_ct_similarity`, that the same kind of failure has happened before, how similar it is to prior failures, and approximately how many times it has happened (saturating at the buffer capacity of 4). The `vsa_rw_strength` scalar represents the intensity of that repetition in [0, 1], and the K_RW coefficient converts it into a small but nonzero explore pressure during each window tick. This is not adaptation; it is pressure. The pressure is the minimum required signal for future adaptation mechanisms to distinguish "first failure" from "recurring failure" -- a distinction that any self-correcting system must be able to make. Day 41 encodes that distinction without acting on it. Day 42 will begin to act.

The six-stage pipeline is now: detect (S_real = spike AND TE high AND PC high) → persist (window, 3 ticks) → interpret (pattern guidance, high/low sim) → commit (micro-strategy, 2 ticks) → detect repetition (continuity buffer, 4-event history) → apply pressure (repetition weighting, K_RW scaled by similarity). This pipeline is fully internal, fully local to the BG scoring path, and fully subordinated to every existing Day 39 and earlier control signal. It is the structure of a self-correcting behavioral loop without the self-correction itself: the structure is present; the adaptation mechanism that would close the loop is not yet implemented. Day 42's trajectory formation will begin to close it.
