## 1. Abstract

A diagnostic script was developed to evaluate whether the Ikigai neural architecture maintains stable excitation-inhibition balance as network size increases. Simulations were executed at three population scales: N=100 (baseline), N=200, and N=400 neurons. Each simulation ran for 1000 ticks with deterministic seed control (seed=42). External measurement of the EI ratio revealed systematic scaling: N=100 produced EI mean 2.44, N=200 produced 3.64, and N=400 produced 5.79. These values increased approximately linearly with excitatory neuron count. However, the internal EIBalanceTracker reported EI ratio 1.04 across all conditions, indicating a measurement mismatch between the internal controller and the true network state. Root cause analysis identified that the EI formula used raw spike counts rather than population-normalized firing rates. Because excitatory neuron count scales with N while only two inhibitory neurons exist in the architecture, the numerator grew proportionally with population size while the denominator remained approximately fixed. No architectural changes were implemented on Day 9. The diagnostic established that the EI measurement system required correction before scale validation could proceed.

## 2. Introduction

### 2.1 Objective

Day 8 classified the system as a Validated Metastable Homeostatic Regime based on perturbation testing at the baseline network size (N=100). Day 9 extends this validation by testing whether the architecture remains stable under population scaling. Biological cortical circuits maintain balanced excitation and inhibition across regions of varying neuron count (Renart et al., 2010). If the Ikigai architecture is biologically faithful, its EI regulatory mechanisms should produce approximately constant EI ratio regardless of how many neurons participate in the network.

### 2.2 Methodology

A standalone diagnostic script (`scripts/day9/scale_test.py`) was written to extract class definitions from `ikigai.py` via AST parsing and instantiate networks at three population sizes. Extra neurons at N=200 and N=400 were added with sparse random wiring into the existing cortex topology. No parameters were retuned for different network sizes. The script recorded excitatory spike counts, inhibitory spike counts, EI ratio, conflict density, regional energy levels, and Big Five trait values every 10 ticks for 1000 ticks per session.

## 4. Root Cause Analysis

### 4.1 The EI Formula

The EI ratio in `EIBalanceTracker.update()` was computed as:

```
r = (exc_count + 1) / ((inh_count + 1) * 4.0)
```

The `exc_count` variable is the total number of excitatory neurons that fired on a given tick. The `inh_count` variable is the total number of inhibitory neurons that fired. The `* 4.0` factor was intended to model the biological 4:1 excitatory-to-inhibitory population ratio in cortex.

### 4.2 Why This Fails at Scale

At N=100 with approximately 10% firing rate, `exc_count` averages approximately 10. At N=400, `exc_count` averages approximately 40. Meanwhile, `inh_count` remains 0 or 1 because the architecture contains only two inhibitory neurons regardless of network size.

The ratio therefore scales as:

```
N=100: r = 11 / (1 * 4) = 2.75
N=400: r = 41 / (1 * 4) = 10.25
```

The `* 4.0` correction is a static constant that does not adjust for population size.

### 4.3 The Same Bug in the Main Loop

The main simulation loop at line 3695 contained an identical formulation:

```
bio_ii = ii * 4.0
c_ei_ratio = c_spikes / bio_ii
```

This fed the EI homeostasis controller with a population-dependent signal, causing the gain correction terms (`_dev_c`, `_dev_l`, `_dev_m`) to respond to population size rather than to actual excitation-inhibition imbalance.

An additional workaround had been introduced to compensate:

```
_ei_nscale = math.sqrt(_N_ei / 100.0)
k1_ei = 0.002 / _ei_nscale
```

This `sqrt(N/100)` scaling factor partially masked the raw-count bug by weakening the controller gain at larger N, rather than fixing the underlying measurement.

## 5. The Correct Metric

### 5.1 Firing Rates

The biologically correct EI metric uses firing rates rather than raw counts:

```
exc_rate = exc_count / n_exc
inh_rate = inh_count / n_inh
```

Firing rates are already normalized by population size. A 10% excitatory firing rate produces `exc_rate = 0.10` regardless of whether the network contains 100 or 400 neurons. Similarly, a 50% inhibitory firing rate (one of two neurons fires) produces `inh_rate = 0.50` regardless of N.

### 5.2 Population Compression

The simulation contains only two inhibitory neurons. Each inhibitory neuron represents a compressed population of biological interneurons. The ratio `exc_rate / inh_rate` alone must be scaled by a compression factor to reflect this representation:

```
r = exc_rate / (inh_rate * population_scale)
```

The `population_scale` parameter models the effective inhibitory contribution of each compressed interneuron node.

---

## 6. Conclusion

The EI scaling diagnostic revealed that the Ikigai architecture itself was not unstable. The measurement system was flawed. The EI controller was optimizing a metric that conflated population size with neural dynamics. Correcting the metric to use population-normalized firing rates was identified as the primary task for Day 10. No architectural changes were implemented on Day 9.
